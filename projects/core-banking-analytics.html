Part 1

Data Preparation & Transformation

Before analysis could begin, the raw dataset required significant preparation across four key tasks: date standardisation, missing value audit, derived column creation, and deduplication.

1

Date Field Standardisation

Eight date columns were identified and standardised to DD/MM/YYYY format using Format Cells. A mixed alignment issue was discovered — left-aligned cells indicated text-stored dates rather than true date values.

⚠ Issue Encountered After applying date formatting, some cells remained left-aligned, confirming they were stored as text rather than true Excel date values.
✔ Resolution Applied Text to Columns with MDY format was applied to all eight date columns. After conversion, all cells became right-aligned — confirming true Excel date storage across the entire dataset.
2

Missing Value Audit

A full dataset audit using Ctrl+A → F5 → Special → Blanks confirmed zero blank cells across all 5,000 rows and 40 columns. Treatment formulas were documented for completeness.

3

Derived Column Creation

Three new columns were added to enrich the dataset: Customer Full Name (concatenation of first and last name), Transaction Category (High Value vs Normal based on $100K threshold), and Loan Risk Flag (High Risk vs Low Risk based on loan approval status).

=A2&" "&B2 → Customer Full Name
=IF(Q2>100000,"High Value","Normal") → Transaction Category
=IF(Z2="Rejected","High Risk","Low Risk") → Loan Risk Flag
4

Deduplication

A composite key was created by joining Customer ID and Transaction ID. Remove Duplicates was run on this key — confirming all 5,000 records were unique with no duplicates in the dataset.

Part 3

Lookup & Reference Functions

A dynamic search panel was built on a dedicated Lookups sheet, allowing retrieval of customer and loan data by simply updating the ID input cells. Three lookup methods were demonstrated and validated against each other.

V

VLOOKUP — Email Retrieval by Customer ID

Retrieved customer email address from Column 9 of the full dataset using an exact match on Customer ID.

=VLOOKUP(B3,core_banking!A:AN,9,FALSE)
X

XLOOKUP — Modern Lookup with Error Handling

Retrieved the same email address using XLOOKUP, which does not require the lookup column to be leftmost and handles missing values natively without IFERROR.

=XLOOKUP(B3,core_banking!A:A,core_banking!I:I,"Not Found")
I

INDEX / MATCH — Loan Amount by Loan ID

Returned the loan amount for a given Loan ID using a two-stage formula: MATCH locates the row, INDEX retrieves the value.

=INDEX(core_banking!U:U,MATCH(B4,core_banking!T:T,0))

All three functions returned consistent, matching results — confirming formula accuracy across methods.

Business Recommendations

Actionable Insights for the Business

Three priority recommendations emerge directly from the pivot table findings, each targeting a measurable business outcome.

📋

Review Loan Approval Criteria

A 32.6% rejection rate significantly exceeds the 20–25% industry benchmark. The business should audit rejection reasons to identify whether the policy is overly restrictive or the applicant pool requires better pre-screening.

⏱️

Reduce Complaint Resolution Time

With 50% of complaints pending and an average resolution time of 124.7 days, a dedicated complaint resolution task force or SLA policy is recommended. Target: reduce average resolution to under 60 days within one quarter.

📈

Capitalise on Deposit Volume Trends

Deposits represent the highest transaction type at $4.26M (33.9%). The business should investigate the July transaction peak to determine whether it is seasonal or driven by a specific product — and plan campaigns accordingly.
